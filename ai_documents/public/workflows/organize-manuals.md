# Workflow: Organize Manuals

Guide to organizing multiple manuals into a project.

## Overview

1. Create a project
2. Add chapters
3. Add manuals to chapters
4. Compile project

## Prerequisites

- At least one existing manual
- Clear idea of desired structure

## Step 1: Create Project

**Navigate to Projects page**
- Click **Projects** in sidebar (`nav-projects`)
- Or go to `/dashboard/projects`

**Create new project**
- Click **Create Project** button (`create-project-btn`)
- Enter project name (e.g., "User Guide")
- Add description (optional)
- Click **Create**

## Step 2: Add Chapters

**Open project**
- Click on your new project to open it

**Create chapters**
- Click **Add Chapter**
- Enter chapter title (e.g., "Getting Started")
- Repeat for each chapter needed
- Drag chapters to reorder

**Example structure:**
```
User Guide
├── Chapter 1: Introduction
├── Chapter 2: Basic Features
├── Chapter 3: Advanced Topics
└── Chapter 4: Troubleshooting
```

## Step 3: Add Manuals

**Open a chapter**
- Click on the chapter to expand it

**Add manuals**
- Click **Add Manual**
- Select manual from your library
- Manual is linked to the chapter
- Repeat for additional manuals

**Organize**
- Drag manuals to reorder within chapter
- Move manuals between chapters if needed

## Step 4: Compile Project

**Start compilation**
- Click **Compile** button
- Select output format

**Configure options**
- Include table of contents
- Add chapter dividers
- Apply template

**Download**
- Click **Download**
- Compiled document saves to your computer

## Tips

**Planning**
- Sketch out structure before creating
- Use descriptive chapter names
- Keep logical flow in mind

**Content**
- One topic per manual
- Consistent formatting across manuals
- Review compiled output before sharing

**Maintenance**
- Changes to manuals reflect in project
- Re-compile after manual updates
- Keep project structure simple

This comprehensive guide will walk you through integrating our API into your application. Whether you're building a web app, mobile application, or backend service, this tutorial provides everything you need to get started.

Before You Begin
Make sure you have an active account and have generated your API key from the dashboard. You'll need this key for authentication.

Overview

Our REST API provides programmatic access to all platform features. It follows standard HTTP conventions and returns JSON responses.

Key Features

  • RESTful architecture with predictable resource-oriented URLs
  • JSON request and response bodies
  • Standard HTTP response codes and authentication
  • Comprehensive error messages for debugging
Step 1

Install the SDK

First, install our official SDK for your programming language. We support Node.js, Python, Ruby, and PHP.

SDK Installation Diagram
npm install @videomanual/sdk
# or
pip install videomanual-sdk
# or
gem install videomanual
# or
composer require videomanual/sdk
Step 2

Configure Authentication

Initialize the SDK with your API key. Store your key securely as an environment variable. 

import VideoManual from '@videomanual/sdk';

const client = new VideoManual({
  apiKey: process.env.VIDEOMANUAL_API_KEY
});

// Verify connection
const account = await client.account.get();
console.log('Connected as:', account.name);
Step 3

Upload Your First Video

Use the upload endpoint to send video files to the platform for processing.

Upload Flow Diagram
const video = await client.videos.upload({
  file: fs.createReadStream('tutorial.mp4'),
  title: 'Getting Started Tutorial',
  description: 'Introduction to our product'
});

console.log('Video uploaded:', video.id);
console.log('Status:', video.status);

Error Handling

The API uses conventional HTTP response codes. Codes in the 2xx range indicate success, 4xx indicate client errors, and 5xx indicate server errors.

Best Practice
Always implement retry logic with exponential backoff for transient errors (5xx codes). This ensures your integration is resilient to temporary network issues. 
try {
  const manual = await client.manuals.get(manualId);
  return manual;
} catch (error) {
  if (error.status === 404) {
    console.error('Manual not found');
  } else if (error.status === 429) {
    console.error('Rate limit exceeded');
  } else {
    console.error('API error:', error.message);
  }
  throw error;
}
